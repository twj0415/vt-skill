# UI 质量规范

## 基础要求

前端实现不只要求功能可用，也要保证界面清晰、交互稳定、状态完整。

## 状态完整性

常见 UI 状态必须考虑：

- 默认状态
- loading
- hover
- focus
- active
- disabled
- error
- empty
- success
- selected

## 可访问性

- 可点击元素要有明确交互反馈。
- 表单控件要有关联 label 或可理解的占位说明。
- 键盘可操作的组件不要破坏 focus。
- 颜色不能作为唯一信息来源。
- 文本对比度要足够。

## 响应式

- 移动端、平板、桌面布局都要考虑。
- 不让内容在小屏溢出。
- 表格、卡片、弹窗、导航在窄屏要可用。

## 视觉一致性

- 间距、圆角、阴影、字体、颜色遵循项目现有设计体系。
- 不混用多套风格。
- 不使用泛滥的渐变、阴影和特效。

## 组件库

如果项目使用组件库，优先遵循组件库模式，不重复造基础组件。自定义样式不要破坏组件库交互和可访问性。
