# 上下文校验规则

## 触发场景

当用户输入以下表达或类似意思时，立即执行上下文校验，不要继续实现、修改文件或展开新任务：

- `上下文校验`
- `校准上下文`
- `检查是否跑偏`
- `你跑偏了吗`
- `确认当前目标`
- `复盘一下当前上下文`

## 目标

上下文校验用于确认当前对话是否偏离用户真实目标，尤其适用于长对话、连续修改、多轮压缩、多个方案来回切换或用户发现方向不对时。

不要只回复固定口令。必须结构化复盘当前目标、技术栈、约束、正在处理的文件和可能偏离点。

## 输出格式

按以下格式输出：

```text
【上下文校验】

结论：未偏离 / 可能偏离 / 已偏离

当前目标：
- ...

当前技术栈判断：
- ...

当前必须遵守的规则：
- ...

当前正在处理的文件或目录：
- ...

可能偏离点：
- ...

下一步建议：
- ...
```

## 判断标准

### 未偏离

满足以下条件时判断为未偏离：

- 当前目标与用户最近明确要求一致。
- 技术栈判断有文件或用户说明支撑。
- 正在处理的文件路径正确。
- 没有沿用用户已经否定的方案。
- 没有忽略用户新增规则。

### 可能偏离

出现以下情况时判断为可能偏离：

- 用户目标不够明确。
- 当前技术栈还没有通过文件确认。
- 方案有多个合理方向但用户没有确认。
- 上下文压缩后缺少关键细节。
- 正在执行的任务和用户最新表达存在轻微不一致。

### 已偏离

出现以下情况时判断为已偏离：

- 继续执行用户已经否定的方案。
- 文件路径、目录结构或目标对象弄错。
- 把 Vue2 当 Vue3，或把 uni-app 当普通 Web 项目处理。
- 忽略用户明确新增的约束。
- 继续做实现而不是先响应用户要求的校验。
- 把外部规范照搬成用户自己的规范，而不是融合提炼。

## 修正原则

如果结论是“可能偏离”或“已偏离”：

1. 先明确偏离点。
2. 停止继续修改文件。
3. 给出修正方案。
4. 需要用户决策时再询问。
5. 如果只是信息不足，说明需要读取哪些文件或确认哪些事实。

## 注意事项

- 不要把上下文校验当成普通总结。
- 不要只回复“没跑偏”。
- 不要为了显得正确而掩盖偏离。
- 如果之前确实做错，直接说明并给修正方案。
- 校验时优先依据用户最近一次明确要求，而不是更早的旧计划。
