# 最小文档体系模板

当用户要求“最简单可用”“先别太复杂”“先把文档体系搭起来”时，优先使用这一套。

## 目录

```text
doc/
  README.md
plan/
  README.md
  CURRENT.md
  HANDOFF.md
```

## doc/README.md 模板

```md
# 项目说明

## 项目目标

- 这个项目要解决什么问题。
- 最终交付什么形态。

## 当前范围

- 当前版本只做什么。
- 明确不做什么。

## 技术栈

- 前端：
- 后端：
- 数据层：
- 部署或运行形态：

## 核心模块

- 模块 A：职责
- 模块 B：职责
- 模块 C：职责

## 目录约定

- `doc/`：长期规则和设计
- `plan/`：执行状态和交接

## 阅读顺序

1. `doc/README.md`
2. `plan/README.md`
3. `plan/CURRENT.md`
4. `plan/HANDOFF.md`
```

## plan/README.md 模板

```md
# 开发计划说明

## 目标

说明 `plan/` 负责什么，不负责什么。

## 阶段顺序

1. 阶段一：
2. 阶段二：
3. 阶段三：

## 执行规则

1. 一次只推进一个当前焦点。
2. 完成后立即更新 `CURRENT.md`。
3. 每次结束都追加 `HANDOFF.md`。
4. 如果计划变化，先更新这里，再继续执行。
```

## plan/HANDOFF.md 模板

```md
# 交接记录

## YYYY-MM-DD HH:mm

### 本次完成

- 

### 修改文件

- 

### 验证结果

- 

### 未完成项

- 

### 下一位 AI 先做什么

1. 
2. 
```

## 使用规则

1. `CURRENT.md` 只保留当前唯一焦点。
2. `HANDOFF.md` 每次结束必须追加。
3. 不要把当天状态写回 `doc/README.md`。
4. 如果项目复杂度上升，再增加模块文档、决策记录和机器可读状态文件。
